# Phase 6 — Polish

## Goals
- Color themes: By Kind (default), By Size (heat map), Monochrome
- Search / filter: sidebar field + treemap highlight ring for matches
- Show / hide hidden files toggle (dotfile filter in both layout engine and sidebar)
- Keyboard shortcuts: Cmd+R rescan, Cmd+[ back navigation
- Drill-down animation: new layout fades in with `.easeInOut(0.18s)`
- Hover tooltip: full path + formatted byte count via `.help()`
- App icon: programmatically generated PNGs + `AppIcon.appiconset` asset catalog

## Files Changed

### `Sources/DiskMapper/Layout/TreemapLayout.swift`
- Added `public var nodeFilter: (@Sendable (FileNode) -> Bool)?`
- Updated `init` to accept `nodeFilter`
- In `layout()`, applies filter to `node.children` before squarifying

### `Sources/DiskMapperApp/Views/FileKindColor.swift`
- Added `enum ColorTheme: String, CaseIterable, Identifiable` — `.byKind`, `.bySize`, `.monochrome`
- Added `color(for:theme:sizeRatio:)` overload
  - `.byKind` → delegates to existing `color(for:)`
  - `.bySize` → log-scale ratio mapped from cool blue (hue 0.65) → warm red (hue 0.0)
  - `.monochrome` → greyscale brightness from 0.22 (small) to 0.77 (large)

### `Sources/DiskMapperApp/Views/TreemapViewModel.swift`
- Added `@Published var colorTheme: ColorTheme = .byKind`
- Added `@Published var searchText: String = ""`
- Added `@Published var showHiddenFiles: Bool = false`
- Added computed `searchMatchIDs: Set<String>` — IDs whose names match `searchText`
- Added `rescheduleLayout()` — re-uses `canvasSize` (called by hidden-files toggle)
- `scheduleLayout()` now builds a `TreemapLayout(nodeFilter:)` with the hidden-file filter
- New layout results are applied with `withAnimation(.easeInOut(duration: 0.18))`
- **Compiler note**: the `nodeFilter` closure must be constructed *before* the `Task {}` block
  and cannot be a ternary with a `nil` branch — use `var … = nil` + `if !showHiddenFiles { … }`
  to avoid "expression is ambiguous" errors in Swift 5.9

## Known Limitations
- The `ColorTheme` picker triggers a Canvas redraw (via `objectWillChange.send()`) but does not
  re-run the layout, so the colour change is instant on the next hover/selection event.
  A full redraw fires automatically on next interaction or window resize.
- Hidden-files toggle resets to the scan root (drill-down depth is preserved via `rescheduleLayout`
  which reuses `displayRoot`, not `rootNode`).
